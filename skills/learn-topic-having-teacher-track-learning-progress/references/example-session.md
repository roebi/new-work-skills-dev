# Example Session — "Git Basics"

This is an annotated example showing how the teaching loop runs.

---

## Phase 1 — Topic Discovery

**Teacher:** Welcome! What topic would you like to learn today?

**Learner:** Git basics

*→ Teacher stores topic = "git-basics", creates `git-basics-learning-process.md`*

---

## Phase 2 — Curriculum Design (Teacher internal step)

Teacher searches web for "git basics tutorial beginner", designs 5-level plan,
writes `git-basics-learning-process.md` with all levels at `[ ] Not started`.

---

## Phase 3 — Loop Iteration 1 (Level 1)

**Teacher (TEACH):**
> Git is a distributed version control system. It tracks changes to files over
> time so you can recall specific versions later. Key terms:
> - **Repository (repo):** a project folder tracked by Git
> - **Commit:** a saved snapshot of changes
> - **Branch:** an independent line of development

**Teacher (ASK):**
> In your own words: what is a Git commit?

**Learner:** A commit is like saving a checkpoint of my files.

**Teacher (EVALUATE):** ✅ Exactly right! A commit is a snapshot — your "checkpoint" analogy is perfect.

*→ Teacher updates file: Level 1 Q1 answered, status → `[~] In progress`*

---

## Loop Iteration 2 (Level 1 continued)

**Teacher (ASK):**
> What command do you use to create a new Git repository in a folder?

**Learner:** git start?

**Teacher (EVALUATE):** Close! The command is `git init`. It initialises a new empty repository in the current folder.

*→ Teacher updates file: Level 1 Q2 answered with correction note*

---

## Loop Iteration 3 (Level 1 → 2)

**Teacher:** You've got the fundamentals of Level 1 down. Let's move to Level 2!

*→ Teacher updates file: Level 1 → `[✓] Mastered`, Level 2 → `[~] In progress`*

---

## Phase 4 — End

**Learner:** learned

**Teacher:** Great session! Your progress has been saved to `git-basics-learning-process.md`. Come back any time to continue. Keep learning! 🎓

*→ Teacher writes Session Summary to file.*
